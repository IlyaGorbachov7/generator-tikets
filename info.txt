------------- Весь этот проект нужно разбить на MVC-подпроекты-подмодули -------------------

    Model является TicketGenerator
Так как оно может быть использовано где угодно.

    View является всё то, что связано с изображением
Это может быть как Desktop-приложение как и Web-прилепине

    Controller является местом связи межу Model и View
Он получает данные из View.
Вызывает Model для совершения какого-то действия, забрасывая данные из View,
необходимые для model-и.
Model обрабатывает данные - возвращет результат.
Controller возвращает результат во View.

-- Это разбиение очень ярко отражает архитектурный замысел и назначения каждого модуля
Центром всея - является Model. Именно под нее стоится View и Controller
