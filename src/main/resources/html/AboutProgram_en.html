Exam Ticket Constructor

The program allows you to generate a list of exam tickets
presented in the form of a .docx file.
The list of questions must be provided with the file extension .docx

To read the list of questions from the file, you need to use the tag: <S>

This tag defines a list of questions formatted according to certain requirements

It has a starting tag: <S> and an ending tag: <\S>

The following attributes can be applied to the starting tag:

  • “l” – indicates difficulty level. Data type – integer;
  • “n” – indicates the topic name. Data type – string;
  • “r” – indicates the number of repetitions. Data type – integer;
An example of placing attributes inside the starting tag looks
like this:

<S>n=Programming Systems; l=23; r=3;>

Specifying these attributes is not mandatory if the "Continuous List" method
of generating tickets is used. In that case, you can just specify the starting
tag: <S>. However, if the method is "Grouped by Difficulty" or "Grouped by Topics,"
these attributes are required.

Requirements for attribute formatting

  • The attribute name is specified first;
  • The “=” symbol follows the attribute name. There can be spaces between the attribute name
    and the “=” symbol;
  • The attribute value follows the “=” symbol, and its type is determined by the attribute itself.
    There can be spaces between the “=” symbol and the value;
  • The “;” symbol is expected after the attribute value. There can be spaces between the value and the “;” symbol;
  • All or some attributes can be inside the starting tag in any order. The attributes specified inside
    the starting <S> tag will apply to all questions in the list.

Requirements for paragraph formatting for tag presence

  • Tags are written in uppercase letters;
  • The paragraph must be centered;
  • The paragraph should not contain anything other than the <S> or <\S> tag.

Requirements for formatting the question list

  • The list of questions should be placed between the starting <S> tag and the closing <\S> tag;
  • Each question must be numbered;
  • After the opening <S> tag, a numbered question list should follow in a new paragraph,
    and after the list, the closing <\S> tag should be placed in a new paragraph.

You can specify these same attributes for an individual question.
To do this, you need to include an element at the beginning of the text of the numbered question: {...},
inside which all previously described attributes can be specified.

An example of placing attributes inside a specific question looks
like this:

1.{ l=23; r=3; }Some question content….

Requirements for question content.

  • The question may contain graphic information: images,
    mathematical equations, formulas.
  • The question content extends from the start of its numbering to the next question
    or to the closing tag.
  • Any type of formatting can be applied to the text of the question:
    colors, bold, italics, strikethrough, underline, superscript, subscript, and text effects such as outline, shadow, reflection, and others.
  • All layout modes can be applied to an image: inline, text wrapping.
    In the latter case, the image will relate to the question it is tied to
    and will have the same placement as specified when formatting it in the question.

In this case, attention must be paid to the attributes when formatting the document
and the desired method for generating tickets. Each method of generating tickets
requires the presence of its attribute, and if this attribute is missing from the starting tag: <S>...> or above a specific question: {...},
this list/question will be ignored, leading to other handling of such exceptions.
